### Post-gate catch — call-site wiring (caught pre-eyeball)
All deterministic gates were green, but they did **not** prove the live behaviour: `buildAiMessageBubble` gained `typing`/`userName`/`userAvatarUrl` and the test pumps the function **directly** with those args — but the screen's own call site (`_buildMessageList` itemBuilder) **still called the 2-arg form**. So live: user bubbles would render `BbAvatar(name:'')` → "?" (not owner initials), and the streaming bubble would show the static `'...'` text (typing dots never wired). Classic "added the capability, didn't wire the use site"; the full-screen pump that *would* catch it is blocked by `enhancedAuthProvider` (StateNotifier). **Caught by the operator's insistence on the live streaming check** before commit. **Fixed:** `_buildMessageList` now reads the owner from `enhancedAuthProvider.userModel` (mirrors `OwnerAppDrawer`) and threads `userName`/`userAvatarUrl` to user bubbles + `typing: streamingText.isEmpty` to the streaming bubble (dots → streamed text on first chunk). Re-verified: `analyze` 0, seam 14/14. **Lesson:** an `@visibleForTesting` widget seam proves the widget, not its wiring — the call site needs a live eyeball (or a provider-overridden full-screen pump).

**[T1] dispatch test — DEFERRED (balloons).** Pumping `AiAssistantScreen` to assert the wiring needs ProviderScope overrides for `aiChatConsentProvider` + `aiChatsProvider` + `aiChatNotifierProvider` (StateNotifier fake + `AiChatState` fixture) + `enhancedAuthProvider` (StateNotifier fake + `UserModel`/`EnhancedAuthState` fixtures) **plus** the 3 providers `OwnerAppDrawer` reads in every layout's Scaffold (`overbookingConflictCountProvider`/`pendingBookingsCountProvider`/`unreadNotificationsCountProvider`) and their transitive deps — the documented full-owner-screen pump blocker. Per operator time-box → deferred (don't rat-hole). ⚠️ **The call-site wiring (`_buildMessageList` → owner identity + `typing`) is EYEBALL-GATED-ONLY**: no automated test guards it, so a future edit to `_buildMessageList` can silently re-break it with all gates green. **Re-eyeball the streaming/avatar path on any change to that method.** Cheaper compile-time alternative (not applied — operator decision): make `typing`/`userName`/`userAvatarUrl` **required** named params so a 2-arg call fails to compile (would have caught this bug at `analyze`).
