## SSRF review

Method: read `icalSync.ts` validator + pin + redirect handler; offline 16-case replay of `validateIcalUrl`; live 8-case smoke via deployed `syncIcalFeedNow` on bookbed-dev with `bookbed-test@bookbed.io` ID token + throwaway feed doc under `SEED_test_owner_property_01`.

**Validator (`isPrivateOrUnsafeIp` + `validateIcalUrl`):**
- DNS lookup runs ONCE via `dns.lookup(hostname, {all: true, verbatim: true})` (line 155).
- ALL returned addresses checked (line 167-178). Reject if ANY private.
- First resolved address pinned + returned (line 183).
- Coverage: RFC1918 (10/8 + 172.16/12 + 192.168/16), 127/8, 169.254/16, 0/8, CGNAT 100.64/10, multicast 224/4 reserved 240/4, IPv6 ::1, ::, fe80::, fc00::/7, ff00::/8, IPv4-mapped IPv6 (decimal form).

## The regression (commit `1c3d6985`)

Live smoke on bookbed-dev exposed: 7 attack vectors blocked correctly, **1 legit URL (www.example.com) failed with `ERR_INVALID_IP_ADDRESS: Invalid IP address: undefined`**.

**Root cause:** Node 18+ enables `autoSelectFamily` by default. `net.createConnection` calls our pinned `lookup` callback with `options = {hints: 1024, all: true}` and expects the **array form**:

```js
callback(null, [{address, family}, ...]);
```

PR #514 returned the 3-arg form:

```js
callback(null, pinnedAddress, pinnedFamily || 4);
```

Node treated the address string as a malformed array → `ERR_INVALID_IP_ADDRESS`. **Every legit feed fetch would have failed after merge** — scheduledIcalSync 15-min cron would mark all feeds `status: error` silently.

CI tests passed because `https.get` was mocked end-to-end — the lookup callback was never invoked against real `net.connect`.

**Fix:**

```ts
const pinnedFam = pinnedFamily || 4;
const requestOptions: https.RequestOptions = pinnedAddress
  ? {
      lookup: (
        _hostname: string,
        options: {all?: boolean} | undefined,
        callback: (...args: any[]) => void,
      ) => {
        if (options && options.all) {
          callback(null, [{address: pinnedAddress, family: pinnedFam}]);
        } else {
          callback(null, pinnedAddress, pinnedFam);
        }
      },
    }
  : {};
```

**Regression test:** `test/icalSync.test.ts` "pinned lookup dispatches array form when options.all === true" captures the `options` arg passed to mocked `https.get` and asserts the lookup callback dispatches correctly for both signatures via synthetic options object — no real network needed. Gates this class going forward.

## Deferred follow-ups (NOT blockers)

1. **Hex IPv4-mapped IPv6 regex hole** in `isPrivateOrUnsafeIp` (icalSync.ts:38). Regex `/^::ffff:(\d+\.\d+\.\d+\.\d+)$/i` matches only **decimal** form. Hex form (e.g. `::ffff:a9fe:a9fe` = 169.254.169.254) NOT caught. Direct URL-literal attack is blocked indirectly because `URL.hostname` returns bracketed form which `dns.lookup` ENOTFOUNDs. But **DNS rebinding to a hex IPv4-mapped IPv6 address bypasses** — attacker DNS returns `::ffff:a9fe:a9fe` directly. Threat is narrow (owner-supplied URL, attacker-controlled DNS, GCP dual-stack socket translation behavior dependent). Harden regex: also catch `/^::ffff:[0-9a-f]{1,4}:[0-9a-f]{1,4}$/i` and translate hex words to dotted v4. → **SF-052 candidate**.
2. **stripePayment.ts** still has local `isAllowedReturnUrl` (lines 80-120). Equivalent logic to `utils/returnUrlValidation.ts`. → separate cleanup PR.
3. **F-NEW-09 timestamp server-bind** — client uses `Timestamp.fromDate`; bind to `request.time` once client switches to `FieldValue.serverTimestamp` (TODO acknowledged inline in rules).
