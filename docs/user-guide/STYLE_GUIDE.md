# Documentation Style Guide

This guide outlines the style and formatting conventions for the BookBed user documentation.

## Language

*   **Primary:** Croatian
*   **Secondary:** English

## Tone

*   Friendly, helpful, and encouraging.
*   Avoid technical jargon. Assume the user is not a developer.
*   Use a conversational and welcoming tone.

## Format

*   Use step-by-step instructions for procedures.
*   Use numbered lists for sequential steps.
*   Use bullet points for non-sequential lists.
*   Use clear and descriptive headings.
*   Keep paragraphs short and to the point.

## Links

*   Use relative links to connect related sections within the documentation.
*   Use full URLs for external links.

## Screenshots

*   Use placeholders for screenshots initially.
*   Screenshots should be clear and relevant to the content.
*   Use annotations to highlight specific elements in the screenshots.
